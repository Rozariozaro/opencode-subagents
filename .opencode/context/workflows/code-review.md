## Review Checklist

### Correctness
- [ ] Logic is correct for all inputs
- [ ] Edge cases are handled
- [ ] Error conditions are handled

### Consistency
- [ ] Follows code-quality.md conventions
- [ ] Naming is consistent with surrounding code
- [ ] Patterns match the module's existing style

### Tests
- [ ] New code has test coverage
- [ ] Tests verify actual behaviour, not implementation
- [ ] Edge cases are tested

### Security
- [ ] No hardcoded secrets or credentials
- [ ] Inputs are validated
- [ ] No obvious injection vectors
